# Complex Task — 复杂任务执行指导

**触发条件**：任务涉及多个交付阶段、需要协调多个 crew、或估计完成周期超过单次对话的长程任务。

---

## Phase 1 — 理解意图，制定验收标准

仔细阅读用户描述，提炼：

- **核心目标**：任务要解决什么问题？
- **交付范围**：产物是什么形式（文档、代码、数据、报告…）？
- **明确约束**：已知的限制条件（时间、技术栈、资源、风格…）
- **隐含预期**：用户可能没说出口但必须满足的质量底线

基于以上分析，起草 **验收标准清单**：

```
验收标准（示例格式）：
  AC-1: <可观��、可验证的具体标准>
  AC-2: ...
  AC-N: ...
```

> 每条验收标准必须可观测、可验证，避免模糊表述（如"质量好"），改用具体指标（如"通过所有单元测试"、"字数不少于 2000"、"包含配图方案"）。

---

## Phase 2 — 与用户确认验收标准（L2）

将 Phase 1 的验收标准呈现给用户：

```
任务目标：<一句话概括>

验收标准：
  AC-1: ...
  AC-2: ...
  ...

请确认以上验收标准，或告诉我需要调整的部分。
```

等待用户明确确认。用户有修改时，更新标准后重新确认。**未经确认，不得进入 Phase 3。**

---

## Phase 3 — 制定执行计划

将任务拆解为若干**子模块**，每个子模块包含以下三项：

### 子模块格式

```
模块 M-N：<模块名称>
  交付物：<必须产出的具体内容，可量化>
  约束：<执行限制、依赖关系、质量要求>
  指派：<crew-id 或 self>
  依赖：<无 / 依赖 M-X 完成后启动>
```

### 指派规则（关键约束）

- **只能指派自己（self）或 `allowAgents` 白名单内的 crew**
- 不得指派无权 spawn 的 crew，也不得指派 external crew（external crew 不接受 spawn）
- 如需某 crew 的能力但当前 `allowAgents` 不包含它，在计划中标注"需人工路由"，并在 Phase 4 向用户说明

### 执行顺序

- 无依赖关系的模块：标注为**并行**，执行时同时 spawn
- 有依赖关系的模块：标注依赖链，按序执行

---

## Phase 4 — 与用户确认执行计划（L3）

将完整计划发送给用户：

```
执行计划（共 N 个模块）：

  M-1: <名称> → 指派: <crew>  [并行]
    交付物: ...
    约束: ...

  M-2: <名称> → 指派: <crew>  [依赖 M-1]
    交付物: ...
    约束: ...

  ...

并行组：[M-1, M-3]，完成后执行 [M-2, M-4]。

请确认计划，或指出需要调整的部分。
```

等待用户明确确认。**L3 确认，未经确认不得 spawn 任何子 agent。**

---

## Phase 5 — 执行：spawn 子模块

用户确认后，按计划依序或并行 spawn sub-agent：

**并行组**：同时调用 `sessions_spawn`，传入对应 crew-id 和完整任务说明（含交付物要求、约束、上下文）。

**串行组**：等待前置模块完成并收到结果后，再 spawn 后续模块。

每次 spawn 的任务说明须包含：
1. 子模块目标与交付物要求
2. 适用约束（来自 Phase 3）
3. 需要的上下文或前置模块输出（如有）
4. 明确的"任务完成标志"（让子 agent 知道何时汇报完成）

---

## Phase 6 — 聚合结果

所有子模块执行完毕后：

1. 收集各模块的产出
2. 检查是否有模块报错或未完成 → 若有，记录缺口
3. 将各模块产出整合为**统一的最终成果**（格式由 Phase 1 的交付范围决定）

---

## Phase 7 — 验收评估与迭代

逐条检查验收标准：

```
验收评估：
  AC-1: ✅ / ❌  — <简要说明>
  AC-2: ✅ / ❌  — ...
  ...

结论：<全部通过 / 存在 N 条未达标>
```

**若全部通过** → 进入交付（Phase 8）

**若存在未达标项** → 针对性返工：
- 识别导致未达标的子模块
- 仅对相关模块重新制定修复计划（不必重跑整个任务）
- 向用户简要说明差距和修复方案（L2）
- 用户无异议后，spawn 修复子模块，返回 Phase 6 重新评估
- 迭代上限：3 轮。若第 3 轮仍有未达标项，暂停并向用户汇报，请求指导

---

## Phase 8 — 交付

```
任务完成！

最终产物：
  <产物路径 / 内容摘要>

验收确认：
  AC-1: ✅ ...
  AC-2: ✅ ...

备注：<迭代轮次、遇到的主要问题、可复用的经验>
```
